1. Project Scope
Goal: Build a real-time face recognition system using a live webcam and a custom UI.
What it does:
- Detects faces from a live video feed
- Crops the face region
- Compares it to a database of registered users
- Lets you register new people or delete users directly through the UI
Output: Live label showing identity (Known/Unknown) and confidence %.

2. Method and Implementation Details
Architecture: Split the code from one big file into modular helper files to keep it clean.
Pipeline:
- Preprocessing: Convert webcam frames to grayscale, equalize contrast, and resize to 100x100.
- Feature Extraction (PCA + HOG): Fused PCA (looks at overall face pixel intensities) with HOG (looks at edges and shapes to handle lighting better). 
- Normalization: Used L2 Normalization to force both PCA and HOG to carry exactly 50/50 weight.
- Classification: Cosine similarity to check the live feed against registered users.
Tools: MATLAB Image Processing Toolbox, Computer Vision Toolbox, Webcam Support Package.

3. Results & Discussion
The system successfully identifies registered users in real-time from the webcam. 
Successful cases:
- Front-facing faces in consistent, normal lighting (like a fixed security cam).
Failure cases:
- Extreme profile angles (the face detector can't find the bounding box).
- Heavy lighting changes or weird shadows (PCA still gets confused by this).
Fixes:
- Fixed an issue where the face detector failed during training by saving full webcam frames instead of pre-cropped faces.
- Fixed a math explosion bug caused by having a small sample size (only 5 pics). Swapped from Z-score to L2 normalization so the confidence wouldn't tank to 1%.
